```yaml
name: brain-cache

```

**Body content — write as direct instructions to Claude (the skill IS the prompt):**

1. **What brain-cache does** (2-3 sentences max): Local embeddings via Ollama, vector index of codebase, retrieves only relevant code so Claude sees less but better context. Emphasize: "save tokens, save money" — this is the pitch.

2. **Tool routing table** — a simple markdown table:

| Query type | Tool | NOT this |
|-----------|------|---------|
| Locate a function, type, or symbol | `search_codebase` | `build_context` |
| Understand how specific code works across files | `build_context` | file reads |
| Diagnose brain-cache failures | `doctor` | -- |
| Reindex the project | `index_repo` | -- |

3. **Per-tool guidance** — for each of the 3 tools:

  **search_codebase (locate code):**
  - Call `mcp__brain-cache__search_codebase` to find functions, types, definitions by meaning not keyword
  - Use for: "Where is X defined?", "Find the auth middleware"
  - Do NOT use for understanding how code works — use build_context once you have located the symbol

  **build_context (understand behavior):**
  - Call `mcp__brain-cache__build_context` with a focused question about how specific code works
  - Use for: "How does X work?", "What does this function do?", debugging unfamiliar code paths
  - Do NOT use for locating symbols (use search_codebase first)
  - Do NOT use for architecture overviews — just answer from what you know

  **index_repo (reindex):**
  - Call `mcp__brain-cache__index_repo` only when user explicitly asks to reindex, or after major code changes
  - Do not call proactively

4. **General rule:** "Use brain-cache tools before reading files or using Grep/Glob for codebase questions. They return better, token-efficient results."

5. **Status line:** "brain-cache displays cumulative token savings in the Claude Code status bar. After tool calls you will see `brain-cache down-arrow{pct}% {n} saved` — this confirms cost savings are working."

6. **doctor tool:** "Call `mcp__brain-cache__doctor` when any brain-cache tool fails or returns unexpected results."

Do NOT include: trace_flow, explain_codebase, file watching, compression, cohesion, or any v2.0+ features.
Do NOT use emojis in the file.
