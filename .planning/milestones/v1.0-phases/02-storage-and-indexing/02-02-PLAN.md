Create `src/services/chunker.ts`:

1. **CJS/ESM shim at top of file** (CRITICAL — tree-sitter is a CJS native addon):
   ```typescript
   import { createRequire } from 'node:module';
   import { extname } from 'node:path';
   import { childLogger } from './logger.js';
   import type { CodeChunk } from '../lib/types.js';

   const _require = createRequire(import.meta.url);
   const Parser = _require('tree-sitter');
   const { typescript: tsLang, tsx: tsxLang } = _require('tree-sitter-typescript');
   const pythonLang = _require('tree-sitter-python');
   const goLang = _require('tree-sitter-go');
   const rustLang = _require('tree-sitter-rust');
   ```

2. Export `LANGUAGE_MAP` — a `Record<string, object>` mapping file extensions to tree-sitter language objects:
   ```
   '.ts' -> tsLang, '.tsx' -> tsxLang, '.mts' -> tsLang, '.cts' -> tsLang,
   '.js' -> tsLang, '.jsx' -> tsxLang, '.mjs' -> tsLang, '.cjs' -> tsLang,
   '.py' -> pythonLang, '.pyi' -> pythonLang,
   '.go' -> goLang,
   '.rs' -> rustLang
   ```
   NOTE: Use TypeScript grammar for JS files — it is a superset and parses JS correctly.

3. Export `CHUNK_NODE_TYPES` — a `Record<string, Set<string>>` mapping language categories to extractable node types:
   ```
   typescript: Set(['function_declaration', 'function_expression', 'arrow_function', 'generator_function_declaration', 'class_declaration', 'abstract_class_declaration', 'method_definition'])
   python: Set(['function_definition', 'async_function_definition', 'class_definition'])
   go: Set(['function_declaration', 'method_declaration', 'func_literal'])
   rust: Set(['function_item', 'impl_item', 'closure_expression'])
   ```

4. Create helper `function getLanguageCategory(ext: string): string` that maps extensions to category keys ('typescript', 'python', 'go', 'rust').

5. Create helper `function extractName(node: any): string | null` that reads `node.childForFieldName('name')?.text` or returns null.

6. Create helper `function extractScope(node: any): string | null` that walks `node.parent` looking for class_declaration/class_definition parent and returns its name, or null.

7. Create helper `function classifyChunkType(nodeType: string): 'function' | 'class' | 'method'` that maps tree-sitter node types to the CodeChunk chunkType enum:
   - `class_declaration`, `abstract_class_declaration`, `class_definition`, `impl_item` -> 'class'
   - `method_definition`, `method_declaration` -> 'method'
   - everything else -> 'function'

8. Export `function chunkFile(filePath: string, content: string): CodeChunk[]`:
   a. Get extension via `extname(filePath)`
   b. Look up language in `LANGUAGE_MAP`; if not found, return `[]`
   c. Create parser, set language, parse content
   d. Walk the AST with a recursive generator `walkNodes(node)`
   e. For each node whose type is in the appropriate `CHUNK_NODE_TYPES` set, create a CodeChunk:
      - `id`: `${filePath}:${node.startPosition.row}` (unique enough for indexing)
      - `filePath`: the input filePath
      - `chunkType`: from classifyChunkType
      - `scope`: from extractScope
      - `name`: from extractName
      - `content`: `content.slice(node.startIndex, node.endIndex)`
      - `startLine`: `node.startPosition.row + 1` (1-based)
      - `endLine`: `node.endPosition.row + 1` (1-based)
   f. **Fallback**: If zero chunks extracted and language was found, emit one `file`-type chunk covering the entire content
   g. Log at debug level: `log.debug({ filePath, chunkCount: chunks.length }, 'File chunked');`
   h. Return chunks array

IMPORTANT: Do NOT nest chunks. If a method is inside a class, extract both the class AND the method as separate chunks. Do not skip the class if it contains methods — both are valid semantic units.

IMPORTANT: For arrow functions, only extract top-level or exported arrow functions (depth <= 2 from root). Nested callbacks inside function bodies should not be extracted as separate chunks.
