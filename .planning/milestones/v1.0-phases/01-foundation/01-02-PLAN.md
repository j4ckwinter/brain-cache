Create `src/services/capability.ts` with these exact exported functions:

1. **detectNvidiaVRAM(): Promise&lt;number | null&gt;** — Use `execFile` (NOT `exec`) from `node:child_process` via `promisify`. Call `nvidia-smi` with args `['--query-gpu=memory.total', '--format=csv,noheader,nounits']`. Set `{ timeout: 3000 }` option to avoid hanging (per Pitfall 2 in research). Parse stdout: `parseInt(stdout.trim().split('\n')[0], 10)`, convert MiB to GiB via `Math.round(mib / 1024)`. Return `null` on any error (command not found, parse failure, timeout). Per D-03.

2. **detectAppleSiliconVRAM(): Promise&lt;number | null&gt;** — Return `null` immediately if `process.platform !== 'darwin'`. Use `execFile` to call `system_profiler` with args `['SPHardwareDataType', '-json']` with `{ timeout: 3000 }`. Parse JSON output, extract `data.SPHardwareDataType[0].physical_memory` (format: "16 GB"). Also check `data.SPHardwareDataType[0].chip_type` contains "Apple M" to confirm Apple Silicon (Intel Macs should return null — RAM is not VRAM on Intel). Use regex `match(/^(\d+)\s*GB/i)` on the memory string. Return `null` on any error. Per D-04.

3. **classifyVRAMTier(vramGiB: number | null): VRAMTier** — Pure function. If `vramGiB === null || vramGiB < 2` return `'none'`. If `vramGiB < 8` return `'standard'`. Otherwise return `'large'`. Per D-01.

4. **selectEmbeddingModel(tier: VRAMTier): string** — Pure function. Map: `none` -> `'nomic-embed-text'`, `standard` -> `'nomic-embed-text'`, `large` -> `'mxbai-embed-large'`. Per D-01.

5. **readProfile(): Promise&lt;CapabilityProfile | null&gt;** — Read `PROFILE_PATH` with `readFile` from `node:fs/promises`. Parse JSON, validate with `CapabilityProfileSchema.safeParse()`. Return parsed data on success, `null` on any error (file missing, invalid JSON, schema validation failure). Import `PROFILE_PATH` from `../lib/config.js`.

6. **writeProfile(profile: CapabilityProfile): Promise&lt;void&gt;** — Create `GLOBAL_CONFIG_DIR` with `mkdir({ recursive: true })`. Write `JSON.stringify(profile, null, 2)` to `PROFILE_PATH`. Import both from `../lib/config.js`.

7. **detectCapabilities(): Promise&lt;CapabilityProfile&gt;** — Orchestrates detection: try `detectNvidiaVRAM()` first, if null try `detectAppleSiliconVRAM()`. Determine `gpuVendor` based on which returned a value ('nvidia', 'apple', or 'none'). Call `classifyVRAMTier()` with the detected VRAM. Call `selectEmbeddingModel()` with the tier. Return a `CapabilityProfile` object with `version: 1`, `detectedAt: new Date().toISOString()`, and `platform: process.platform`.

Use `childLogger('capability')` from logger.ts for any debug/info logging.

Create `tests/services/capability.test.ts` — Mock `node:child_process` via `vi.mock('node:child_process')` to control nvidia-smi and system_profiler outputs. Mock `node:fs/promises` for profile read/write tests. Use a temp directory for file I/O tests. Test all behavior items listed above.

  <action>
Create `src/services/ollama.ts` with these exact exported functions:

1. **isOllamaInstalled(): Promise&lt;boolean&gt;** — Use `execFileAsync('which', ['ollama'])` on Unix. Wrap in try/catch: success = true, error = false. On `process.platform === 'win32'`, try `execFileAsync('where', ['ollama'])` instead. Per D-05.

2. **isOllamaRunning(): Promise&lt;boolean&gt;** — Use `fetch('http://localhost:11434')`. Return `res.ok` on success. Return `false` on any error (ECONNREFUSED, timeout, etc.). Per D-06.

3. **startOllama(): Promise&lt;boolean&gt;** — Use `spawn('ollama', ['serve'], { detached: true, stdio: 'ignore' })` followed by `child.unref()`. Poll `isOllamaRunning()` every 500ms for up to 5 seconds (10 attempts). Return `true` on first successful poll, `false` if all 10 attempts fail. Log to stderr via `childLogger('ollama')`. Per D-06.

4. **pullModelIfMissing(model: string, onProgress?: (status: string) => void): Promise&lt;void&gt;** — Import the default export from the `ollama` npm package. Call `ollama.list()` and check if any model name starts with the given model string. If model exists, return early. If missing, call `ollama.pull({ model, stream: true })` which returns an AsyncGenerator. Iterate with `for await`, calling `onProgress?.(progress.status)` for each progress event. Default progress output: `process.stderr.write(\`Pulling ${model}: ${progress.status}\\n\`)`. Per D-07.

5. **getOllamaVersion(): Promise&lt;string | null&gt;** — Use `execFileAsync('ollama', ['--version'])`. Parse stdout to extract version string. Return `null` on error.

Import `childLogger` from `../services/logger.js` for all logging.

Create `tests/services/ollama.test.ts` — Mock `node:child_process` for isOllamaInstalled and getOllamaVersion. Mock global `fetch` for isOllamaRunning. Mock the `ollama` npm package for pullModelIfMissing. For startOllama, mock both `spawn` and `fetch`. Test all behavior items listed above.

IMPORTANT: Do NOT use `console.log` anywhere. All output goes through the pino logger or `process.stderr.write` (only for interactive progress during model pull per research Open Question 3).
