Create `src/workflows/init.ts` with a single exported `runInit()` function that orchestrates Phase 1 setup:

```
async function runInit(): Promise<void>
```

**Init workflow sequence (per D-02, D-05, D-06, D-07, D-08, D-12):**

1. Print to stderr: `"brain-cache: detecting hardware capabilities..."`
2. Call `detectCapabilities()` to get GPU info, VRAM tier, embedding model
3. Print to stderr: `"GPU: {gpuVendor} ({vramGiB} GiB) | Tier: {vramTier} | Model: {embeddingModel}"`
4. If `vramTier === 'none'`, print warning to stderr: `"Warning: No GPU detected. Embeddings will run on CPU (slower). Consider using a machine with a GPU for faster indexing."`
5. Check `isOllamaInstalled()`:
   - If false: print to stderr: `"Error: Ollama is not installed.\n\nInstall Ollama:\n  macOS: brew install ollama\n  Linux: curl -fsSL https://ollama.com/install.sh | sh\n\nThen run: brain-cache init"`. Call `process.exit(1)`.
6. Check `isOllamaRunning()`:
   - If false: print to stderr: `"Ollama is not running. Starting..."`. Call `startOllama()`.
   - If startOllama returns false: print to stderr: `"Error: Could not start Ollama. Run 'ollama serve' manually, then retry 'brain-cache init'."`. Call `process.exit(1)`.
7. Get Ollama version via `getOllamaVersion()` and include in profile.
8. Call `pullModelIfMissing(profile.embeddingModel)` — this auto-pulls with progress output to stderr.
9. Call `writeProfile(profile)` to persist to `~/.brain-cache/profile.json`.
10. Print to stderr: `"brain-cache initialized successfully.\n  Profile: ~/.brain-cache/profile.json\n  Embedding model: {model}\n  VRAM tier: {tier}"`

**ALL output via `process.stderr.write()` — ZERO output on stdout (per D-16).**

Create `src/cli/index.ts` as the Commander CLI entry point for brain-cache (per D-17).

```typescript
#!/usr/bin/env node
import { Command } from 'commander';

const program = new Command();

program
  .name('brain-cache')
  .description('Local AI runtime — GPU cache layer for Claude')
  .version('0.1.0');

program
  .command('init')
  .description('Detect hardware, pull embedding model, create config directory')
  .action(async () => {
    const { runInit } = await import('../workflows/init.js');
    await runInit();
  });

program
  .command('doctor')
  .description('Report system health: GPU, VRAM tier, Ollama status')
  .action(async () => {
    const { runDoctor } = await import('../workflows/doctor.js');
    await runDoctor();
  });

program.parse();
```

Key constraints:
- Use dynamic `import()` for workflow modules (lazy loading, keeps startup fast per Commander best practice)
- The `#!/usr/bin/env node` shebang is for reference; tsup adds it via `banner.js` config
- Program name is `brain-cache` (per D-17)
- Version is `0.1.0`
- No `console.log` anywhere — workflows handle all stderr output
- Per D-12: commands other than `init` should check for profile existence. The doctor workflow already handles this internally (reads profile, exits if missing). Future commands added in later phases will use the same pattern.
