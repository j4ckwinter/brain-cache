# Plan 09-01: Concurrent file I/O and streaming chunk pipeline

<objective>
Restructure the index workflow so file reads run with a concurrency limiter (up to 20 concurrent) and chunks are embedded and stored in batches as they are produced, rather than accumulating all chunks in memory before embedding begins. After this plan, indexing is measurably faster on multi-file repos and memory usage is bounded regardless of repo size.
</objective>

<must_haves>
- File reads use a concurrency limiter with a configurable limit (default 20), not sequential `for` + `await`
- Chunks are embedded and flushed to LanceDB in batches as they are produced — `allChunks` is no longer accumulated before the embed loop
- `FILE_READ_CONCURRENCY` constant is exported from `src/lib/config.ts`
- Progress logging still reports chunking and embedding progress to stderr
- Token counting (raw and chunk) is still computed and reported in the summary
- All existing tests pass (`npm test`)
</must_haves>

2. Replace the sequential file-reading loop and the separate embed loop with a combined pipeline. The approach:

   a. Build a concurrency-limited file processor using a native Promise pool pattern (no npm dependency). The pool processes up to `FILE_READ_CONCURRENCY` files at a time. For each file: read with `readFile`, count raw tokens, call `chunkFile`, and collect the chunks into a shared pending batch buffer.

   b. Whenever the pending buffer reaches `DEFAULT_BATCH_SIZE` chunks, flush the batch: embed with `embedBatchWithRetry`, build `ChunkRow[]`, call `insertChunks`, clear the batch, and update progress counters.

   c. After all files are processed, flush any remaining chunks in the buffer as a final batch.

**Correction** — the flush-during-read interleaving is complex because embed calls are async and the concurrency pool must not race on `pendingChunks`. A cleaner approach:

Use a two-phase streaming design within each "wave" of concurrent reads:

```ts
  // Step 7+8: Concurrent file read + streaming embed pipeline
  let totalRawTokens = 0;
  let totalChunkTokens = 0;
  let totalChunks = 0;
  let processedFiles = 0;
  let processedChunks = 0;

  // Process files in groups of FILE_READ_CONCURRENCY.
  // After each group, flush accumulated chunks through embed+store.
  for (let groupStart = 0; groupStart < files.length; groupStart += FILE_READ_CONCURRENCY) {
    const group = files.slice(groupStart, groupStart + FILE_READ_CONCURRENCY);
    const groupChunks: CodeChunk[] = [];

    // Read files concurrently within the group
    const results = await Promise.all(
      group.map(async (filePath) => {
        const content = await readFile(filePath, 'utf-8');
        const rawTokens = countChunkTokens(content);
        const chunks = chunkFile(filePath, content);
        return { rawTokens, chunks };
      })
    );

    // Collect results (sequential to maintain deterministic ordering)
    for (const result of results) {
      totalRawTokens += result.rawTokens;
      groupChunks.push(...result.chunks);
    }
    processedFiles += group.length;
    totalChunks += groupChunks.length;

    if (processedFiles % 10 === 0 || groupStart + FILE_READ_CONCURRENCY >= files.length) {
      process.stderr.write(`brain-cache: chunked ${processedFiles}/${files.length} files\n`);
    }

    // Embed and store this group's chunks in DEFAULT_BATCH_SIZE batches
    for (let offset = 0; offset < groupChunks.length; offset += DEFAULT_BATCH_SIZE) {
      const batch = groupChunks.slice(offset, offset + DEFAULT_BATCH_SIZE);
      const texts = batch.map((chunk) => chunk.content);
      totalChunkTokens += texts.reduce((sum, t) => sum + countChunkTokens(t), 0);
      const vectors = await embedBatchWithRetry(profile.embeddingModel, texts);

      const rows: ChunkRow[] = batch.map((chunk, i) => ({
        id: chunk.id,
        file_path: chunk.filePath,
        chunk_type: chunk.chunkType,
        scope: chunk.scope,
        name: chunk.name,
        content: chunk.content,
        start_line: chunk.startLine,
        end_line: chunk.endLine,
        vector: vectors[i],
      }));

      await insertChunks(table, rows);
      processedChunks += batch.length;
      process.stderr.write(
        `\rbrain-cache: embedding ${processedChunks}/${totalChunks} chunks (${Math.round((processedChunks / totalChunks) * 100)}%)`
      );
    }
  }
  process.stderr.write('\n');
  process.stderr.write(
    `brain-cache: ${totalChunks} chunks from ${files.length} files\n`
  );
```

This replaces the old `allChunks` accumulation pattern. The `allChunks` variable is removed entirely.
