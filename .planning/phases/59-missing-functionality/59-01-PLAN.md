<objective>
Wire edge graph traversal into trace retrieval so that `build_context` in trace mode expands results by following call edges from matched chunks to their callees.

Purpose: Trace queries currently only return vector-search matches. By following stored call edges one hop out, trace results include the functions that matched code calls into, producing richer hop chains for debugging and flow understanding.

Output: `expandByEdges` function in retriever.ts, wired into buildContext.ts for trace mode, with unit tests.
</objective>

1. Early return `[]` if `maxHops <= 0` or `seedChunks.length === 0`
2. Build `seenIds = new Set(seedChunks.map(c => c.id))`
3. For each seed chunk, call `queryEdgesFrom(edgesTable, chunk.id)`

5. Cap to first `MAX_EDGES_PER_CHUNK` edges per seed chunk (slice after filter)
6. For each remaining edge, build SQL predicate: `file_path = '${escapeSqlLiteral(edge.to_file)}' AND name = '${escapeSqlLiteral(edge.to_symbol)}'`
7. Query `chunksTable.query().where(predicate).limit(1).toArray()`
8. If row found and id not in seenIds, add to seenIds and push to expanded array as RetrievedChunk with:
   - `id: r.id`, `filePath: r.file_path`, `chunkType: r.chunk_type`
   - `sourceKind: r.source_kind === 'history' ? 'history' : 'file'`
   - `scope: r.scope ?? null`, `name: r.name ?? null`
   - `content: r.content`, `startLine: r.start_line`, `endLine: r.end_line`
   - `fileType: r.file_type ?? 'source'`, `similarity: 0.5`
9. Return expanded array
