  <action>
    **DEBT-03 — Ollama host configuration:**

    In `src/services/ollama.ts`:

    1. Add a helper function near the top (after imports):
    ```typescript
    function getOllamaHost(): string {
      return process.env.OLLAMA_HOST ?? 'http://localhost:11434';
    }
    ```

    2. In `isOllamaRunning()` (line 28), replace the hardcoded URL:
    - Change: `const res = await fetch('http://localhost:11434');`
    - To: `const res = await fetch(getOllamaHost());`

    3. Export getOllamaHost so other modules can use it if needed:
    `export function getOllamaHost(): string {`

    Note: The `ollama` npm package (imported as `import ollama from 'ollama'`) already respects `OLLAMA_HOST` env var internally for its API calls (embeddings, list, pull). The only place we use raw `fetch` against the Ollama server is `isOllamaRunning()` — that's the one that needs fixing.

    **SEC-01 — API key log redaction:**

    In `src/services/logger.ts`:

    1. Add a `redact` configuration to the pino options object. Change:
    ```typescript
    export const logger = pino(
      { level: resolveLevel() },
      pino.destination(2)
    );
    ```
    To:
    ```typescript
    export const logger = pino(
      {
        level: resolveLevel(),
        redact: {
          paths: [
            'apiKey',
            'api_key',
            'secret',
            'password',
            'token',
            'authorization',
            'ANTHROPIC_API_KEY',
            'OPENAI_API_KEY',
            '*.apiKey',
            '*.api_key',
            '*.secret',
            '*.password',
            '*.token',
            '*.authorization',
            '*.ANTHROPIC_API_KEY',
            '*.OPENAI_API_KEY',
          ],
          censor: '[Redacted]',
        },
      },
      pino.destination(2)
    );
    ```

    This ensures that if any log call accidentally passes an object containing API keys, the values are replaced with `[Redacted]` before being written to stderr.

    Update tests:

    - In `tests/services/logger.test.ts`: add a test that logs an object with `apiKey: 'sk-ant-secret'` and verifies the output contains `[Redacted]` not `sk-ant-secret`.
