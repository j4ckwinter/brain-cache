    **CLI catch wrapper (HARD-01 completion):**

    In `src/cli/index.ts`, wrap `program.parse()` (currently the last line) in an async IIFE with a catch handler:

    Replace `program.parse();` with:
    ```typescript
    (async () => {
      await program.parseAsync();
    })().catch((err: unknown) => {
      const message = err instanceof Error ? err.message : String(err);
      process.stderr.write(`Error: ${message}\n`);
      process.exit(1);
    });
    ```

    Note: Commander's `program.parseAsync()` is used instead of `program.parse()` because the action handlers are async. `parseAsync()` returns a Promise that rejects if an action handler throws, which is exactly what we need to catch workflow errors.

    **MCP entry point (HARD-01 — keep process.exit):**
    The `main().catch()` block at lines 271-274 of `src/mcp/index.ts` already correctly catches errors and calls process.exit(1). This IS an entry point, so process.exit is appropriate here. No change needed beyond the version fix.
