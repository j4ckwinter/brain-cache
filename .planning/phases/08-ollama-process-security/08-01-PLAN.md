# Plan 08-01: Harden Ollama process lifecycle

<objective>
Fix detached Ollama process management so brain-cache never leaves orphaned `ollama serve` processes and never spawns a duplicate instance when one is already running. The current `startOllama` function spawns a detached/unreffed child with no PID tracking, no pre-spawn running check, no signal-handler cleanup, and no kill-on-timeout — all of which are required by SEC-02.
</objective>

<must_haves>
- `startOllama` checks `isOllamaRunning()` immediately before spawning and returns early if Ollama is already up (prevents double-spawn)
- `child.pid` is captured before `child.unref()` and included in all log messages (timeout warn, success info)
- On timeout, `startOllama` sends SIGTERM to the spawned PID before returning `false` (prevents orphan from failed start)
- SIGINT/SIGTERM handlers are registered during the poll loop and removed after it completes (prevents orphan when brain-cache is killed mid-startup)
- `process.exit(1)` calls in `src/workflows/init.ts` are replaced with thrown errors (completes HARD-01 gap)
- All new behaviours have unit tests in `tests/services/ollama.test.ts`
</must_haves>

<tasks>
<task id="08-01-T1">
<title>Add pre-spawn guard and PID tracking to startOllama</title>
<read_first>
- src/services/ollama.ts
- .planning/phases/08-ollama-process-security/08-RESEARCH.md (section 3.1, 3.3)
</read_first>
<action>
In `src/services/ollama.ts`, rewrite `startOllama()` as follows:

1. At the top of the function, call `isOllamaRunning()`. If it returns `true`, log `'Ollama is already running, skipping spawn'` and return `true` immediately. Add an inline comment explaining this reduces the TOCTOU race window (but does not eliminate it entirely — Ollama itself handles EADDRINUSE safely).

2. After the `spawn()` call, capture `const pid = child.pid` before calling `child.unref()`.

3. Register one-time signal handlers for SIGINT and SIGTERM that call `process.kill(pid, 'SIGTERM')` inside a try/catch (ignore ESRCH). Store references to the cleanup function so they can be removed later.

4. In the polling loop, no changes to the loop logic itself.

5. After the loop — on BOTH the success and timeout paths — remove the SIGINT/SIGTERM handlers using `process.removeListener`.

6. On the timeout path (after the loop, when returning `false`):
   - Kill the spawned process: `process.kill(pid, 'SIGTERM')` inside a try/catch.
   - Log with `log.warn({ pid }, 'Ollama did not start within timeout — killed spawned process (PID: ' + pid + ')')`.

7. On the success path, update the existing info log to include `{ pid }`: `log.info({ pid, attempt: attempt + 1 }, 'Ollama is now running')`.
</action>
<acceptance_criteria>
- `isOllamaRunning` is called at the start of `startOllama` before any `spawn` call
- `child.pid` is captured into a `const pid` variable before `child.unref()`
- `process.once('SIGINT'` and `process.once('SIGTERM'` appear inside `startOllama`
- `process.removeListener('SIGINT'` and `process.removeListener('SIGTERM'` appear after the poll loop
- `process.kill(pid, 'SIGTERM')` appears in the timeout branch
- The timeout log includes `{ pid }` in the structured data
</acceptance_criteria>
</task>
